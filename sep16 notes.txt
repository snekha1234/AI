ğŸ” What it shows:

For each day, you get a box showing distribution of total bills.

The line inside the box = median bill.

The box edges = Q1 (25th percentile) and Q3 (75th percentile).

The whiskers = spread of most bills.

The dots outside whiskers = outliers (very high bills).

What is a Heatmap?

A heatmap is a graphical representation of data where values are shown as colors.

Darker/brighter colors â†’ higher values

Lighter colors â†’ lower values

Often used for correlation matrices or to show values in a 2D table.
import seaborn as sns
import matplotlib.pyplot as plt

# Load dataset
iris = sns.load_dataset("iris")

# Select only numeric columns for correlation

# Create heatmap
sns.heatmap(corr, annot=True, cmap="coolwarm", linewidths=0.5)

plt.title("Correlation Heatmap - Iris Dataset")
plt.show()

Calculates correlation between numerical columns.

Values range from -1 to +1:

+1 â†’ perfect positive correlation (as one increases, the other increases).

-1 â†’ perfect negative correlation (as one increases, the other decreases).

0 â†’ no correlation.

corr â†’ gives values to plot.

annot=True â†’ shows numbers inside the squares.

cmap="coolwarm" â†’ color scheme (blue = low, red = high).

linewidths=0.5 â†’ adds small lines between squares for clarity.

What is a Pairplot?

A pairplot is a grid of plots that shows relationships between all numerical variables in a dataset:

Diagonal â†’ histograms (or KDE plots) of each variable.

Off-diagonal â†’ scatterplots between pairs of variables.

Itâ€™s very useful for exploring datasets with multiple numeric features.

Supervised Machine Learning:
What is Supervised Machine Learning?

Supervised learning is a type of machine learning where:

You have a labeled dataset (input features + known output/target).

The model learns the mapping from input â†’ output.

After training, it can predict outputs for new, unseen inputs.

ğŸ‘‰ â€œSupervisedâ€ means the algorithm is guided by correct answers (labels) during training.

ğŸ”¹ Key Components

Features (X) â†’ Input variables (e.g., size of house, number of rooms).

Target (y) â†’ Output variable we want to predict (e.g., house price).

Model â†’ The algorithm (e.g., Linear Regression, Decision Tree).

Training â†’ Model learns from known input-output pairs.

Prediction â†’ Apply model to new data to estimate outputs.

ğŸ”¹ Types of Supervised Learning

Regression (when target is continuous)

Example: Predict house price, temperature, or salary.

Algorithms: Linear Regression, Decision Tree Regression, Random Forest, etc.

Classification (when target is categorical)

Example: Spam vs. Not Spam, Disease vs. No Disease, Sentiment (Positive/Negative).

Algorithms: Logistic Regression, KNN, SVM, Decision Trees, Neural Networks.

ğŸ”¹ Workflow of Supervised Learning

Collect & preprocess data

Split dataset â†’ Train set + Test set

Choose an algorithm

Train the model on training data

Evaluate model performance (accuracy, MSE, precision, recall, etc.)
Tune model parameters (hyperparameters)

Deploy for predictions

Real-Life Examples

Predicting emails â†’ Spam / Not Spam

Predicting bank loan approval

Predicting house prices

Diagnosing diseases from medical data

Classifying images (cat vs dog)

Machine Learning Foundations
1. What is Machine Learning?

Machine Learning is about teaching computers to learn patterns from data instead of being explicitly programmed.

ML models generalize from training data to make predictions on unseen data.

2. Types of Machine Learning

Supervised Learning

Data has input features (X) and labeled outputs (y).

Examples: House price prediction, spam detection.

Algorithms: Linear Regression, Logistic Regression, Decision Trees, SVM.

Unsupervised Learning

Data has only inputs (X), no labels.

Goal: Find hidden structure (clusters, patterns).

Examples: Customer segmentation, anomaly detection.

Algorithms: K-Means, PCA, DBSCAN.

Reinforcement Learning

Agent learns by interacting with environment using trial & error.

Examples: Self-driving cars, AlphaGo.

3. Core Concepts

Dataset â†’ Collection of examples (rows = samples, columns = features).

Features (X) â†’ Input variables.

Target (y) â†’ Output variable (in supervised learning).

Model â†’ Algorithm mapping inputs â†’ outputs.

Training â†’ Learning patterns from data.

Evaluation â†’ Measuring performance (accuracy, RMSE, precision, recall).

Overfitting â†’ Model memorizes training data, fails on new data.

Underfitting â†’ Model too simple, fails to capture patterns.

1. Train-Test Split

When building ML models, we donâ€™t use all data for training.
We split the dataset into:

Training set â†’ Used to train (fit) the model. (~70â€“80%)

Testing set â†’ Used to evaluate performance on unseen data. (~20â€“30%)

from sklearn.model_selection import train_test_split
import pandas as pd

# Example dataset
data = {
    "age": [25,30,35,40,45,50,55,60],
    "salary": [2500,3000,4000,5000,6000,7000,8000,9000],
    "purchased": [0,0,0,1,1,1,1,1]   # Target
}
df = pd.DataFrame(data)

X = df[["age", "salary"]]   # Features
y = df["purchased"]         # Target

# Split: 70% train, 30% test
X_train, X_test, y_train, y_test = train_test_split(
    X, y, test_size=0.3, random_state=42
)

print("Train size:", len(X_train))
print("Test size:", len(X_test))
